Docker Basics

Needs for Docker

  • Matrix from Hell (Managing all the versions of your libraries on your OS).
  • Setting up new environments for new employees is very difficult as each developer had to set up their own environment, running hundreds of commands with correct configurations and versions.
  • Some developers are comfortable with some OS, and ensuring the system runs smoothly across various OS systems (Linux, Windows, etc.) was a big task.

Docker containers run on the underlying OS, so if you're running Docker on a Linux System, you can create containers running Ubuntu, Debian, Fedora, Arch, etc., but not a Windows distribution.

Docker is an LCX type of container.

Docker is not meant to virtualize and run different operating systems and kernels on the same hardware. The main purpose of Docker is to package and containerize applications, ship them, and run them anywhere, anytime, as many times as needed.

Benefits of Docker Against VM

  • Utilizes fewer resources.
  • Uses less space and boots up faster.
  • Has less isolation between resources.

Docker Commands

> docker version

After installing Docker, you can get the Docker version using this command.

> docker run {image_name}

Runs a Docker container from an image. If the image is not found on the host, it will pull it from Docker Hub.

> docker run --name {container_name} {image_name}

Creates a container based on the provided image and assigns it the specified name.

> docker ps

Lists all running containers with basic information. Use -a to list all containers (running or exited).

> docker stop {container_name}

Stops a running container. You must provide either the container ID or name.

> docker rm {container_name}

Removes a container from the system.

> docker images

Lists all Docker images on the system.

> docker rmi {image_name}

Removes an image from the system. Ensure no containers depend on it.

> docker pull {image_name}

Pulls an image from Docker Hub without running a container.

Note: Running docker run ubuntu creates a container that exits immediately as it's not meant to run an OS, but a specific process.

> docker run ubuntu sleep 5

Runs a container and makes it sleep for 5 seconds before exiting.

> docker exec {container_name} {command}

Executes a command on a running container.

> docker run -d myspringbootapp

Runs a service in detach mode.

> docker attach {container_id}

Attaches back to a service running in a container.
