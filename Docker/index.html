Docker Beginner Guide

Needs for Docker

  • Managing library versions on your OS (Matrix from Hell).
  • Setting up new environments for employees is tedious and requires many commands with precise configurations.
  • Handling smooth operation across various OS (Linux, Windows, etc.) is challenging.

Docker Basics

Docker containers run on the underlying OS. For example, on a Linux system, you can create containers running various Linux distributions (Ubuntu, Fedora, etc.), but not Windows. Docker is not for virtualizing different operating systems. Its purpose is to package applications to run anywhere.

Docker vs Virtual Machines

Key Differences:

  • Docker uses fewer resources, less space, and starts faster.
  • Virtual Machines provide better isolation but are resource-heavy.

Docker Commands

Here are essential Docker commands for beginners:

docker version

Displays the Docker version installed on your system.

docker run {image_name}

Runs a container from the specified image. If the image isn't available locally, it pulls it from Docker Hub.

docker ps -a

Lists all containers, including those that have stopped.

docker stop {container_name}

Stops a running container.

docker rm {container_name}

Removes a stopped container from the system.

docker images

Lists all available Docker images on your system.

docker rmi {image_name}

Removes a Docker image from your system.

docker pull {image_name}

Pulls an image from Docker Hub without running it.

Important Notes

Docker containers are designed to run processes, not entire operating systems. If you run a container with a base image (e.g., Ubuntu), it may exit immediately unless a process is specified.
